# 期刊选稿画像助手 (Journal Profile Assistant)

在科研成果撰写与投稿流程中，传统工具仅解决“检索范围”与“学科关键字匹配”的问题，而未能深入揭示**目标期刊底层选稿偏好、理论品味与具体写作范式**。本 Skill 通过工业级工程实践构建数据驱动的偏好画像，打通从客观文献抓取到定制化投稿改造的闭环。

## 🌟 适用场景与核心价值

- **选稿预审**：准确识别目标期刊在研究范式（定量/定性/混研/计算）、样本量级、理论框架上的隐性门槛。
- **对标优化**：针对即将投稿的论文摘要或大纲，提出符合该刊审稿人胃口的标题改造、引言框架对齐与实证补强建议。
- **批量调研**：可在数分钟内完成对某领域内主流顶刊前沿学术发文动态的结构化调研。

## 🏗️ 四层流水线架构 (Pipeline Architecture)

本系统将整个流程清晰拆解为四个解耦的工程模块：

```
+-------------------------------------------------------------------------+
| [Layer 1: 数据抓取层] fetch_papers.py                                     |
|   调用 OpenAlex 开放 API 检索目标期刊，筛选近期高质量摘要并重建文本。               |
+-------------------------------------------------------------------------+
                                    |
                                    v
+-------------------------------------------------------------------------+
| [Layer 2: 结构化提取层] extract_features.py                               |
|   并发调用 LLM 将非结构化摘要转换为 JSON/Pydantic 实体（方法/理论/样本/工具）。 |
+-------------------------------------------------------------------------+
                                    |
                                    v
+-------------------------------------------------------------------------+
| [Layer 3: 统计聚合层] aggregate.py                                        |
|   纯 Python 算法计算分布比例、高频理论矩阵、分析工具热度与被引权重偏好。            |
+-------------------------------------------------------------------------+
                                    |
                                    v
+-------------------------------------------------------------------------+
| [Layer 4: 战略生成层] generate_profile.py                                 |
|   结合统计指标与高引案例，LLM 生成深度《期刊偏好画像与修稿策略报告》。              |
+-------------------------------------------------------------------------+
```

## 📋 快速使用说明

### 1. 环境准备
```bash
# 进入项目目录并安装依赖
pip install -r requirements.txt

# 配置环境变量 (复制 .env.example 为 .env)
cp .env.example .env
```

### 2. 运行 CLI 工具
```bash
# 基础运行：生成指定期刊近期研究画像（抓取最近3年，采样30篇）
python main.py --journal "Strategic Management Journal" --years 3 --max-papers 30

# 对标修稿运行：传入个人论文摘要（或全文草稿），生成个性化修改策略
python main.py --journal "Nature Communications" --user-draft path/to/my_abstract.txt
```

### 3. 输入参数说明
| 参数 | 缩写 | 默认值 | 说明 |
|---|---|---|---|
| `--journal` | `-j` | *(必须)* | 目标期刊标准全称（如 `Computers in Human Behavior`） |
| `--years` | `-y` | `3` | 回溯统计的最长年限（以当前年份往前推算） |
| `--max-papers` | `-m` | `30` | 采样参与分析的有效摘要论文数量上限 |
| `--user-draft` | `-u` | `None` | 用户准备投稿的论文摘要/草稿文本路径（用于深度对标修改建议） |
| `--output` | `-o` | `output/` | 生成最终画像报告 Markdown 文件的保存路径 |

## 📦 输出交付物结构

系统执行完毕后将输出一份完整的 Markdown 报告，核心包含三大区块：
1. **📊 选稿偏好核心画像**：量化展示范式分布饼图数据、偏好理论矩阵、样本阈值底线。
2. **🚩 审稿雷区与避坑指南**：基于近期低频方法/盲区识别最易被 Desk Reject 的写作方式。
3. **🎯 投稿定制化改造策略**：提出标题包装、Abstract 重构提示、结构化建议及补充实验指导。
